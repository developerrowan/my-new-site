Updating the Rest of the Pages

Leave the blog home page without a side bar for now, but update the other pages in the same way, and have them display the hobbies side bar. Here are the steps to repeat in the other pages:

  1. Remove the aside tag from each page.
  2. Add the $sideBar variable and initialize it to a value of hobbies-sidebar.inc.php (declare and initialize it just under the $pageTitle and $pageDescription variables)
  3. Add the IF statement PHP code to the bottom of the page (DON'T use ../ in the path to the sidebar include file)
  4. Test the page in the browser

Finally, you can remove the yellow background color from the MAIN tag in the .css file.

Summary

Note the pattern we are following on our pages:

  • Include the config file first
  • Set page specific variables (we could store these values in a database if we wanted to, which would give us a new level of flexibility)
  • Display the header (which uses the page specific variable settings)
  • Display the page content
  • Display the side bar (if there is one)
  • Display the footer

Our site is becoming more dynamic and flexible (the css code changes based on whether or not there is a side bar shown on the page!) And because we've arranged things using include files, our site is much easier to maintain!
